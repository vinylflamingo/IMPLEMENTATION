A ticket based system for tracking request in a support organization.

This project was created out of a need to solve an issue. At my support job, edits and fixes that needed to be made were all getting dumped into a single email inbox. While this was good for holding them, it did not provide them with notifications or tracking of said request. Using ASP.NET Core 3.1 I created a system in which support agents could submit and track their own tickets, while managers could complete tickets or pass tickets to other agents. Also managers could subscribe to reporting to where what the most troublesome issues were. I used a mostly microsoft tech stack. Some technologies used are Entity Framework, SQL Server, Postgres, Bootstrap, AJAX, and some fun Javascript front end libraries.
